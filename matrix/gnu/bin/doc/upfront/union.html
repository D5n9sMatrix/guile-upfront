Unions







Unions, a low-level feature inherited from the C programming language, allow more than one member to share the same memory area.



Unions are very similar to structs with the following main differences:





  • Unions are defined by the keyword.


  • The members of a are not independent; they share the same memory area.




Just like structs, unions can have member functions as well.



The examples below will produce different results depending on whether they are compiled on a 32-bit or a 64-bit environment. To avoid getting confusing results, please use the compiler switch when compiling the examples in this chapter. Otherwise, your results may be different than mine due to alignment, which we will see in a later chapter.

Unions are not a memory-saving feature. It is impossible to fit multiple data into the same memory location. The purpose of a union is to use the same area for different type of data at different times. Only one of the members can be used reliably at one time. However, although doing so may not be portable to different platforms, members can be used for accessing fragments of other members.

Unions can have as many members as needed. All of the members would share the same memory location.



The fact that the same memory location is used for all of the members can have surprising effects. For example, let's initialize a object by its member and then access its member:



  1. auto u = U(42); // initializing the int member writeln(u.d); // accessing the double member

Depending on the endianness of the microprocessor, the 4 bytes may be arranged in memory as 0|0|0|42, 42|0|0|0, or in some other order. For that reason, the value of the member may appear differently on different platforms.

Dissecting other members


Unions can be used for accessing individual bytes of variables of other types. For example, they make it easy to access the 4 bytes of an IPv4 address individually.



The 32-bit value of the IPv4 address and a fixed-length array can be defined as the two members of a :



  1. union IpAddress { uint value; ubyte[4] bytes; }


The members of that would share the same memory area as in the following figure:





For example, when an object of this is initialized by 0xc0a80102 (the value that corresponds to the dotted form 192.168.1.2), the elements of the array would automatically have the values of the four octets:



  1. import std.stdio; void main() { auto address = IpAddress(0xc0a80102); writeln(address); }

The reverse order of the octets is another example of how accessing different members of a may produce unpredictable results. This is because the behavior of a is guaranteed only if that is used through just one of its members. There are no guarantees on the values of the members other than the one that the has been initialized with.

Examples


Communication protocol


In some protocols like TCP/IP, the meanings of certain parts of a protocol packet depend on a specific value inside the same packet. Usually, it is a field in the header of the packet that determines the meanings of successive bytes. Unions can be used for representing such protocol packets.



The following design represents a protocol packet that has two kinds:



  1. struct Host { // ... } struct ProtocolA { // ... } struct ProtocolB { // ... } enum ProtocolType { A, B } struct NetworkPacket { Host source; Host destination; ProtocolType ; { ProtocolA aParts; ProtocolB bParts; } ubyte[] payload; }

Discriminated union


Discriminated union is a data structure that brings type safety over a regular . Unlike a , it does not allow accessing the members other than the one that is currently valid.



The following is a simple discriminated union type that supports only two types: and . In addition to a to store the data, it maintains a member to know which one of the two members is valid.



  1. import std.stdio; import std.exception; struct Discriminated { private: TypeInfo validType_; union { int i_; double d_; } public: this(int value) { // This is a call to the property function below: i = value; } // Setter for 'int' data void i(int value) { i_ = value; validType_ ; } // Getter for 'int' data int i() const { enforce(validType_ , "The data is not an 'int'."); return i_; } this(double value) { // This is a call to the property function below: d = value; } // Setter for 'double' data void d(double value) { d_ = value; validType_ ; } // Getter for 'double' data double d() const { enforce(validType_ , "The data is not a 'double'." ); return d_; } // Identifies the type of the valid data const(TypeInfo) type() const { return validType_; } } unittest { // Let's start with 'int' data auto var = Discriminated(42); // The type should be reported as 'int' assert(var.type == typeid(int)); // 'int' getter should work assert(var.i == 42); // 'double' getter should fail assertThrown(var.d); // Let's replace 'int' with 'double' data var.d = 1.5; // The type should be reported as 'double' assert(var.type == typeid(double)); // Now 'double' getter should work ... assert(var.d == 1.5); // ... and 'int' getter should fail assertThrown(var.i); }
