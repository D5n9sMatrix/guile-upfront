The loop serves the same purpose as the loop. makes it possible to put the definitions and expressions concerning the loop's iteration on the same line.



Although is used much less than in practice, it is important to understand the loop first. We will see in a later chapter.



The sections of the loop


The loop evaluates the loop condition and continues executing the loop as long as that condition is . For example, a loop to print the numbers between 1 and 10 may check the condition less than 11:



  1. while (number < 11)


Iterating the loop can be achieved by incrementing at the end of the loop:



  1. ++number;


To be compilable as D code, must have been defined before its first use:



  1. int number = 1;


Finally, there is the actual work within the loop body:



  1. writeln(number);


These four sections can be combined into the desired loop as follows:



  1. int number = 1; // ← preparation while (number < 11) { // ← condition check writeln(number); // ← actual work ++number; // ← iteration }

The sections of the loop


The loop brings three of these sections onto a single line. They are written within the parentheses of the loop, separated by semicolons. The loop body contains only the actual work:



  1. for (/* preparation */; /* condition check */; /* iteration */) { /* actual work */ }


Here is the same code written as a loop:



  1. for (int number = 1; number < 11; ++number) { writeln(number); }


The benefits of the loop are more obvious when the loop body has a large number of statements. The expression that increments the loop variable is visible on the line instead of being mixed with the other statements of the loop. It is also more clear that the declared variable is used only as part of the loop, and not by any other surrounding code.



The sections of the loop are executed in the same order as in the loop. The and statements also work exactly the same way as they do in the loop. The only difference between and loops is the name scope of the loop variable. This is explained below.



Although very common, the iteration variable need not be an integer, nor it is modified only by incrementing. For example, the following loop is used to print the halves of the previous floating point values:



  1. for (double value = 1; value > 0.001; value /= 2) { writeln(value); }


Note: The information above is technically incorrect but better captures the spirit of how the loop is used in practice. In reality, D's loop does not have three sections that are separated by semicolons. It has two sections, the first of which consisting of the preparation and the loop condition.



Without getting into the details of this syntax, here is how to define two variables of different types in the preparation section:



  1. for ( int i = 0; double d = 0.5; i < 10; ++i) { writeln("i: ", i, ", d: ", d); d /= 2; }


Note that the preparation section is the area within the highlighted curly brackets and that there is no semicolon between the preparation section and the condition section.



The sections may be empty


All three of the loop sections may be left empty:



  • Sometimes a special loop variable is not needed, possibly because an already-defined variable would be used.
  • Sometimes the loop would be exited by a statement, instead of by relying on the loop condition.
  • Sometimes the iteration expressions depend on certain conditions that would be checked within the loop body.


When all of the sections are emtpy, the loop means forever:



  1. for ( ; ; ) { // ... }


Such a loop may be designed to never end or end with a statement.



The name scope of the loop variable


The only difference between the and loops is the name scope of the variable defined during loop preparation: The variable is accessible only within the loop, not outside of it:



  1. for (int i = 0; i < 5; ++i) { // ... } writeln(i); // i is not accessible here




In contrast, when using a loop the variable is defined in the same name scope as that which contains the loop, and therefore the name is accessible even after the loop:



  1. int i = 0; while (i < 5) { // ... ++i; } writeln(i); // ← 'i' is accessible here


We have seen the guideline of defining names closest to their first use in the previous chapter. Similar to the rationale for that guideline, the smaller the name scope of a variable the better. In this regard, when the loop variable is not needed outside the loop, a loop is better than a loop.
