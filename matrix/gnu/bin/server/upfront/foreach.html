foreach Loop







One of the most common statements in D is the loop. It is used for applying the same operations to every element of a container (or a range).



Operations that are applied to elements of containers are very common in programming. We have seen in the Loop chapter that elements of an array are accessed in a loop by an index value that is incremented at each iteration:



  1. for (int i = 0; i != array.length; ++i) { writeln(array[i]); }


The following steps are involved in iterating over all the elements:



  • Defining a variable as a counter, which is conventionally named as


  • Iterating the loop up to the value of the property of the array


  • Incrementing


  • Accessing the element


has essentially the same behavior but it simplifies the code by handling those steps automatically:



  1. foreach (element; array) { writeln(element); }


Part of the power of comes from the fact that it can be used the same way regardless of the type of the container. As we have seen in the previous chapter, one way of iterating over the values of an associative array in a loop is by first calling the array's property:



  1. auto values = aa; for (int i = 0; i != values.length; ++i) { writeln(values[i]); }


does not require anything special for associative arrays; it is used exactly the same as with arrays:



  1. foreach (value; aa) { writeln(value); }


The syntax


consists of three sections:



  1. foreach (names; container_or_range) { operations }


  • container_or_range specifies where the elements are.


  • operations specifies the operations to apply to each element.


  • names specifies the name of the element and potentially other variables depending on the type of the container or the range. Although the choice of names is up to the programmer, the number of and the types of these names depend on the type of the container.


and


These keywords have the same meaning as they do for the loop: moves to the next iteration before completing the rest of the operations for the current element, and terminates the loop altogether.



with arrays


When using with plain arrays and there is a single name specified in the names section, that name represents the value of the element at each iteration:



  1. foreach (element; array) { writeln(element); }


When two names are specified in the names section, they represent an automatic counter and the value of the element, respectively:



  1. foreach (i, element; array) { writeln(i, ": ", element); }


The counter is incremented automatically by . Although it can be named anything else, is a very common name for the automatic counter.

with number ranges


We have seen number ranges before, in the Slices and Other Array Features chapter. It is possible to specify a number range in the container_or_range section:



  1. foreach (number; 10..15) { writeln(number); }


Remember that 10 would be included in the range but 15 would not be.



with structs, classes, and ranges


can also be used with objects of user-defined types that define their own iteration in loops. Structs and classes provide support for iteration either by their member functions, or by a set of range member functions. We will see these features in later chapters.



The counter is automatic only for arrays

The copy of the element, not the element itself


The loop normally provides a copy of the element, not the actual element that is stored in the container. This may be a cause of bugs.



To see an example of this, let's have a look at the following program that is trying to double the values of the elements of an array:



  1. import std.stdio; void main() { double[] numbers = [ 1.2, 3.4, 5.6 ]; writefln("Before: %s", numbers); foreach (number; numbers) { number *= 2; } writefln("After : %s", numbers); }


The output of the program indicates that the assignment made to each element inside the body does not have any effect on the elements of the container:





That is because is not an actual element of the array, but a copy of each element. When the actual elements need to be operated on, the name must be defined as a reference of the actual element, by using the keyword:



  1. foreach ( number; numbers) { number *= 2; }


The new output shows that the assignments now modify the actual elements of the array:





The keyword makes an alias of the actual element at each iteration. As a result, the modifications through modify that actual element of the container.



The integrity of the container must be preserved


Although it is fine to modify the elements of a container through variables, the structure of a container must not be changed during its iteration. For example, elements must not be removed nor added to the container during a loop.



Such modifications may confuse the inner workings of the loop iteration and result in incorrect program states.
